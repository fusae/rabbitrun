2026 年 3 月 31 日，Claude Code 因为一次 source map 暴露，让外界看到了部分内部源码。这不是一次普通的"大厂翻车"。对关心 AI 的人来说，这次泄漏真正的价值在于：我们第一次看清了，一个顶级 AI agent 到底长什么样。很多人以为，所谓 agent，不过是更强的模型 + 更长的 prompt + 几个工具接口。但源码暴露出的现实完全不是这样。Claude Code 的核心，不是"一个会聊天的模型"，而是一整套围绕模型构建的运行系统。模型很重要，但它只是系统里的中枢，不是系统本身。从整体结构看，Claude Code 至少由七层组成。

## 第一层：入口与控制

Claude Code 不是单一用途的命令行工具。它从设计之初就支持多模式运行——既可以作为交互式终端使用，也能进入非交互执行模式，还能承载远程连接、服务端会话、桥接等场景。这说明它的定位不是"把 Claude 搬进终端"，而是做一个统一的 agent runtime，让同一套核心能力在不同入口复用。

## 第二层：交互控制台

用户看到的是终端界面，但从内部组织看，这个界面更像 agent 的驾驶舱。它不只展示对话，还要管理任务状态、权限状态、工具运行、远程连接、通知、插件、资源和上下文切换。换句话说，Claude Code 的 UI 本质不是聊天窗口，而是能实时操控 agent 的控制台。这意味着 Anthropic 从一开始就没把 agent 当"对话产品"，而是当"可操作系统"来做。

## 第三层：agent 核心循环

这是整套架构最关键的一层。传统聊天产品的逻辑很简单：用户输入，模型输出，结束。但 Claude Code 的逻辑更像持续运转的任务循环。用户给出目标后，系统先组织上下文，让模型决定下一步；如果需要调用工具，就执行工具；工具返回结果后，再喂回模型继续判断；如此反复，直到任务完成。它的核心不是"回答一次问题"，而是"持续推进一个任务"。这也是为什么顶级 agent 的重点不再只是回答质量，而是任务执行能力。

## 第四层：工具调度

从源码最容易看出的事实是：工具不是外挂，工具就是 agent 的骨架。模型负责理解目标、规划步骤、生成决策，但真正与外部世界发生作用的，是读文件、改文件、执行命令、搜索、联网、调用外部服务这些工具。更重要的是，工具不是随意暴露给模型，而是经过统一抽象和调度管理。哪些工具可以并发，哪些必须串行；哪些只读，哪些有副作用——这些不是小细节，而是 agent 从 demo 走向产品的关键。这也解释了为什么很多个人项目的 agent 看起来能跑，但任务一复杂就不稳定：问题不在模型，而在工具系统根本没被当"系统级能力"认真设计。

## 第五层：权限与安全

这次泄漏暴露的另一个信号是：Anthropic 对权限问题极其重视。一个能执行命令、修改文件、联网访问资源的 agent，本质上已经不是"聊天机器人"，而是具备操作能力的软件实体。只要它能动手，就必须被约束。Claude Code 的设计里，权限不是后补的安全补丁，而是从一开始就嵌在执行链路里。系统会考虑哪些行为要询问、哪些可以自动通过、哪些必须拒绝、哪些属于高风险。这说明对顶级 agent 来说，真正的难点不是"让它更敢做事"，而是"让它做事时仍然可控"。没有这层设计，能力越强，风险越大。

## 第六层：状态、记忆与恢复

这是很多人最容易低估、但实际上最接近产品护城河的部分。短对话里，记忆管理问题不明显；但任务持续十几轮、几十轮后，上下文就会膨胀，信息就会丢失，执行就会跑偏。Claude Code 内部显然花了很大力气处理这些问题——会话延续、上下文压缩、任务恢复、历史状态回放、长任务中的信息保真。这些设计说明，在真实世界里，agent 最大的挑战不是"会不会想"，而是"能不能在长时间执行中不失忆、不崩溃、不偏航"。很多人以为大模型产品的核心竞争力是参数规模，但从这类工程实现看，真正拉开差距的往往是状态管理能力。

## 第七层：扩展平台

Claude Code 没把所有能力写死在产品里，而是为外部扩展留了很大空间。技能系统、插件机制、外部协议接入、IDE 桥接、远程会话支持——这些结构共同透露出一个方向：Anthropic 想做的不只是"会帮你写代码的 AI"，而是可以不断接入新能力的 agent 平台。不是封闭产品思路，而是平台思路。今天它接命令行、编辑器和外部工具，明天也可以进一步变成更完整的 AI 工作环境。

---

如果要把这次泄漏的最大启发压缩成一句话：**顶级 AI agent 不是一个更聪明的聊天机器人，而是由模型、工具、权限、状态、记忆和扩展机制共同组成的运行系统。**这件事把很多人对 agent 的误解直接撕开了。过去讨论 agent，常把重点放在 prompt、思维链、函数调用这些"模型附近"的东西上；但 Claude Code 展现的现实是，真正决定 agent 上限的，往往不是模型那一刻说了什么，而是整个系统如何组织模型、约束模型、放大模型、修正模型。模型像大脑，工具像手脚，权限像护栏，状态系统像长期记忆，扩展机制像外接器官。只有这些部分共同工作，AI agent 才可能从"会说"变成"会做"，再从"会做一点"变成"稳定完成复杂任务"。
