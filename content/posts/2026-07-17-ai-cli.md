昨天看到一条消息：飞书开源了 CLI。看到这个消息，我马上想到一件事：我一直想搭一个新媒体管理系统，之前嫌麻烦一直没动手。现在有了飞书 CLI，Agent 能直接操作飞书的表格、日历、文档，这事儿终于能搞了。于是我打开 AI Agent，开始干活。

## CLI 是个啥？

CLI，全称 Command Line Interface，命令行界面。就是那种黑乎乎的屏幕，你敲一行命令，电脑给你吐一行结果。跟它相对的是 GUI（Graphical User Interface，图形用户界面），也就是我们平时点来点去的那些按钮、菜单、弹窗。GUI 是给人用的，CLI 是给程序用的。而在今天，最需要用 CLI 的程序，就是 AI Agent。

## 让 AI 帮我搭建整个系统

装完飞书 CLI（安装过程很简单，后面会说），我直接对 Agent 说："帮我用飞书搭建一个新媒体管理系统。需要这些模块：Agent 开始干活。

### 模块 1：选题库

Agent 先创建了一个飞书 Base，叫"新媒体管理"。然后在里面创建了"选题库"表格，添加了这些字段：• 标题（文本）• 状态（单选：待写/进行中/已完成）• 优先级（单选：高/中/低）• 计划发布日期（日期）• 备注（文本）然后把我本地的选题都导入进去。N条选题，全部导入完成。

### 模块 2：内容日历

Agent 创建了一个飞书日历，叫"内容发布计划"。然后把选题库里有计划日期的选题，自动同步到日历里。这样我打开飞书日历，就能看到什么时候该发什么内容。

### 模块 3：文章管理

Agent 在飞书云盘创建了一个文件夹，叫"文章库"。以后每篇文章都存成飞书文档，放在这个文件夹里。

### 模块 4：发布记录

Agent 在"新媒体管理"这个 Base 里，又创建了一个"发布记录"表格，字段包括：• 文章标题（文本）• 发布平台（多选：微信公众号/知乎/小红书/抖音）• 发布日期（日期）• 阅读数（数字）• 点赞数（数字）• 文章链接（文本）**整个系统，20 分钟搭好了。**

## 真正的价值：系统化管理，而不是零散文件

这套系统搭好之后，我的工作流程变成了这样：**1. 添加选题**我在飞书选题库里添加一条新选题，设置状态为"待写"，填上计划发布日期。Agent 自动把这条选题同步到内容日历。**2. 开始写作**我把选题状态改为"进行中"，Agent 在飞书文档里创建一篇新文章，标题就是选题标题。**3. 完成发布**文章写完后，我把选题状态改为"已完成"，然后在发布记录表格里添加一条记录，填上发布平台、阅读数、点赞数。**整个流程，数据都在云端，Agent 和我看到的是同一份数据。**以前用本地 Markdown：• 我在编辑器里改选题• Agent 读文件、解析、修改、写回• 中间可能出错、格式可能乱• 各模块之间没有联动现在用飞书 CLI：• 我在飞书 GUI 里改选题状态• Agent 调用 CLI 查询，立刻看到最新状态• Agent 添加新选题，我刷新页面就能看到• 选题库、日历、文档、发布记录，全部打通**这才是真正的"系统化管理"。**

## GUI 是给人看的，CLI 是给 Agent 看的

搭建这套系统的过程，让我想明白一件事：**在 AI 时代，软件需要两套界面。**• **GUI**：给人用，点击、拖拽、表单、弹窗• **CLI**：给 Agent 用，命令、参数、结构化输出以前的软件演化路径是：CLI → GUI电脑最早是命令行，后来有了 Windows、Mac 的图形界面，普通人才开始用电脑。现在路径反过来了：GUI → CLI不是回到过去，而是**新用户来了**。这个新用户，叫 Agent。Agent 不会点鼠标，它只能：• 调用 API（需要写代码、处理认证、部署服务）• 跑命令行（一行命令搞定）API 门槛太高，CLI 刚刚好。飞书 CLI 提供了 200+ 命令，覆盖文档、表格、日历、消息、任务……基本上 GUI 能做的，CLI 都能做。更重要的是，它内置了 19 个 Skills，可以直接被 AI 工具调用。不需要额外配置，装完就能用。**这才是 AI 时代的"开放"：让 Agent 能像人一样使用你的平台。**

## 安装很简单

如果你也想试试，安装过程很简单。直接对你的 AI Agent 说：“帮我装一下所有的东西：https://github.com/larksuite/cli/blob/main/README.zh.md”Agent 会自动安装飞书 CLI 和 19 个 Skills。**注意：一定要确认 Skills 也装上了。** 如果没有，手动让 Agent 跑一下：npx skills add larksuite/cli --all -y -g
然后配置登录：lark-cli config
会给你一个二维码链接，复制到浏览器，扫码授权就行。**最后，记得重启你的 Agent！** 这不是热重载能解决的，必须重启。

## 飞书 CLI 能干什么

目前支持的功能：• **文档**：创建、更新、查询飞书文档• **表格**：创建 Base、添加字段、插入记录、查询数据• **日历**：创建日历、添加日程、查询会议• **消息**：发送私信、群消息• **任务**：创建任务、更新状态• **邮件**：发送邮件、查询邮箱• **云盘**：上传文件、管理权限• **Wiki**：创建 Wiki、管理节点基本上 GUI 能做的，CLI 都能做。

## 协作平台开始为 Agent 设计了

飞书这一步，迈得其实挺狠的。传统产品看 DAU、看停留时长，这些指标都要打开 GUI 才算。但当你把 CLI 开源出来，Agent 可以直接调用命令行，用户有时候就不需要打开飞书了。传统口径的日活会掉，但**真正对用户的价值，反而上升了**。这说明飞书想得很清楚：**效率工具，就应该为用户的效率服务。**从 OpenClaw 开始，飞书就在做 Agent 基建平台。现在开源 CLI，是把整个产品压缩成了一套命令行工具，让 Agent 能真正"使用"飞书。**未来，所有的产品，都将有两种形态：**• **GUI**：面向普通用户• **CLI**：面向开发者和 AI

## 写在最后

搭建这套新媒体管理系统之后，我突然意识到一件事：**软件的使用方式，正在发生根本性的改变。**以前我们用软件，是打开界面、点击按钮、填写表单。现在我们用软件，是告诉 Agent 想要什么，Agent 帮我们调用软件。飞书开源 CLI，不只是多了个命令行工具，而是在说：我们准备好了，Agent 时代来吧。接下来我准备让 Agent 自动管理更多事情：每周检查选题库的缺失字段、自动同步文章到各个平台、定期生成数据报表……这些以前需要手动操作或者写代码的事情，现在一句话就能搞定。如果你也在做内容创作、项目管理、团队协作，不妨试试飞书 CLI。
