 我的 OpenClaw（小龙虾）跑在国内云服务器上，自带的 web_search 走的是 Brave Search API——注册要信用卡，而且主要搜英文内容，对国内用户基本没用。后来发现 OpenClaw 支持接入 Kimi (Moonshot) 作为搜索引擎，中文能力强，注册也不需要信用卡。本以为换个 Provider 就完事了，没想到一连踩了三个坑。

## 坑 1：版本太低不支持

我先按文档配置好 Kimi 的 API Key，结果一调用就报错。查了一下，OpenClaw 版本太旧，根本不支持 Kimi Provider。只能升级：npm update -g openclaw
sh scripts/restart.sh
升级完终于能跑了，但搜出来的结果……

## 坑 2：401 报错——Kimi 有两个站点

域名改对了，又报新错：401 Unauthorized。排查半天才发现原因：Kimi 分国内站和国际站，API 域名不一样。站点域名国内站api.moonshot.cn国际站api.moonshot.ai我是在国内 Kimi 平台注册的 Key，但 OpenClaw 文档示例指向的是国际站域名。国内 Key + 国际站 = 401 Unauthorized改对域名就好了。这个坑官方文档里没提，纯靠自己摸索。

## 坑 3：搜索质量惨不忍睹

能搜索 ≠ 搜索好用。试了下，搜个新闻都没搜到，但在 Kimi 网页版搜同样关键词，结果很丰富。问题出在 OpenClaw 调 Kimi 的方式上——参数、接口、返回格式的适配不够好，导致搜索质量大打折扣。

## 终极方案：绕开 web_search，自己调 API

既然内置工具不行，那就直接调 Kimi 原生 API。Kimi 模型有个内置的 $web_search 工具函数，直接调它，搜索效果和网页版一模一样。做法是写一个两轮调用的脚本：#!/bin/bash
# 第一轮：触发搜索，拿到 tool_call_id 和搜索参数
# 第二轮：把搜索结果喂回去，生成最终回答
bash scripts/moonshot-search.sh "你要搜的内容"
核心逻辑：1. 发一轮请求，让模型调用 $web_search 内置工具2. 拿到 tool_call_id 和搜索参数3. 再发一轮请求，把搜索结果喂回去，让模型生成回答然后把它包装成 OpenClaw 的 Skill，设置 override-tools: - web_search，让小龙虾以后有搜索需求直接走这个 Skill。效果立竿见影——同样的关键词，结果质量天壤之别。
折腾 AI 的日常就是这样——你永远不知道下一个坑在哪里，但每次填完，都会更懂一点。如果你也遇到了同样的问题，不用自己再踩一遍坑——我已经把这个搜索方案做成了一个 Skill，发布在 ClawHub 上了。**Moonshot Search** — 在 ClawHub 搜索即可找到，一行命令安装，自动接管搜索功能。clawhub install moonshot-web-search
