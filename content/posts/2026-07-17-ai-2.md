OpenAI的Codex 上了个大更新：Computer Use 功能终于支持 Windows 了。之前这个功能只有 Mac 能用，Windows 用户等了两个月。3 天前版本推送，我终于能试试了。

## 它能干什么

Computer Use 就是让 Codex "看到"你的屏幕，像人一样点击、打字、操作软件。我让它打开 Chrome 帮我查一个 GitHub 仓库的 Release 说明。它自己移动鼠标点了 Chrome，输入了地址，找到 Release 页面，把内容复制回来。前后大概 2 分钟，我没碰过鼠标。我又试了让它操作 Excel，把一列数据按日期排序。它打开了文件，选中列，点了排序按钮，搞定了。还行，简单任务够用。

## 4 个坑

用了几天，有几个问题得知道。**1. 执行的时候你不能用电脑。**鼠标被 Codex 接管，键盘也是它的。你硬要操作，鼠标会被拽来拽去。给它一个几分钟的任务，去倒杯水，回来验收成果，这是目前最合适的用法。**2. 管理员弹窗会卡住。**Windows 那个"你要允许此应用对设备进行更改吗？"的提示，Codex 点不了，得你手动来。**3. 终端操作不了。**Windows Terminal 它搞不定，需要你介入。**4. 不能后台运行。**得保持前台活跃，不能最小化丢一边。

## 好消息

不需要装 WSL，直接用原生 PowerShell 和 Windows Sandbox。对 Windows 用户来说，这点比预想的友好。Codex 现在每周 400 万活跃用户，Windows 版发布前 50 万人排队等。后台运行、截图记录这些应该会跟上。

## 我的判断

现阶段它适合帮你干几分钟的确定性任务——查个资料、整理个文件、跑个测试。复杂的事它还搞不定，别指望它替你搞定一切。
但想想两个月前 Windows 用户连这个功能都没有。从没有到能用，这一步已经迈出来了。
